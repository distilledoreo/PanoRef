# Browser Smoke Coverage

Use the Codex Browser plugin for runtime smoke verification.

1. Launch the app with `npm run dev`.
2. Open the local URL printed by Vite. It usually starts at `http://localhost:3000`, but use the next free port if Vite reports one.
3. If no mode preference is stored, confirm the mode chooser appears after the splash (or immediately if splash already seen): **Build continuity packages** vs **Just view a 360 pano**. Choose Continuity Stage. Open the brand menu, choose **Current Objective**, confirm the modal appears, and dismiss it with **Got it**. Confirm the stage rail is Build → Reference → Shots → Export (no Review). Optionally switch to **Simple 360 Viewer**, import a pano, look around, and **Download current view**.
4. Use the top-right **Open project** folder button to open a saved Continuity Stage JSON file. Confirm the project-opened status appears and the brand menu shows the imported project name. Repeat with an invalid JSON file and confirm the app reports a clear import error instead of silently failing.
5. In **Build**, confirm the primary tray shows the reference primitives and the **More** tool exposes Select, Origin, Snap, and extra primitives. Confirm the viewport starts centered on the set in orbit/select mode. Open **Visibility distance**, move the slider, and confirm the displayed meter value changes and the fog/shroud edge visibly moves; then toggle **Free camera**, drag to look, hold `W` to move, and press `Esc` to return to the default mode. While active, confirm `D`, `S`, `Ctrl`, and other camera keys do not trigger Build shortcuts or random UI actions, while selected-object edit chrome stays hidden and the phone move pad remains unobstructed above the render CTA. Render **Render 360 Reference**. When the step is complete, confirm an advance modal appears with **Continue to Reference** and **Not right now**.
6. In **Reference**, import a styled pano or use the attached reference. Confirm the alignment guide modal appears, dismiss it with **Start checking**, then approve the reference or open **Reference Settings** to use yaw and graybox fade. Verify the advance modal offers **Continue to Shots** only after alignment is confirmed.
7. Open **Shots**. Confirm the UI looks like a simple camera: full-bleed viewfinder, **Still / Video** mode pills, a large center **shutter**, a bottom-left library thumbnail, and a top-right **settings** gear. Advanced tools (PNG, pano match, FOV, delete, etc.) should only appear in Settings.
8. Confirm the viewfinder stays live (fly mode). Reposition the camera, press the shutter to **Capture** — confirm a quiet “Captured” flash, the shot appears in the library thumbnail, and the viewfinder **does not freeze**. Capture again and confirm a **new** gallery shot is created while you keep flying. Hold **W** past the front wall/gate within the set-adjacent safety volume. Confirm Export advance after at least one shot is captured.
9. Switch mode to **Video**, confirm **Length** chips (1s–8s) appear above the shutter, pick a duration, fly an end pose, press shutter to capture end, then press again to export **MP4** when ready. If the browser reports MP4 recording support, confirm the MP4 downloads. If unsupported or export fails, confirm the reason appears directly above the shutter (and remains available in Settings).
10. In **Export**, confirm each shot row includes a media thumbnail and the manifest includes `inputs/cubemap/pz.png` when full pano is enabled, plus `inputs/viewport_clay_motion.mp4`, `inputs/camera_move/clay_start.png`, `metadata/camera_keyframes.json`, and `metadata/camera_move_reference_frames.json` after a camera move has been exported. Confirm `cubemap_visible` paths are absent. Export with **Export Selected Shots** without importing any AI result and confirm the export checkpoint completes for the selected shot.
11. At each workspace, confirm default visual density stays close to the references: stage rail, full-bleed surface, contextual cards only when relevant, primary CTA, and drawer-based advanced controls.
12. Repeat at a phone-sized viewport and confirm modals fit the screen, drawers scroll cleanly, and there is no horizontal overflow from the main canvas.
